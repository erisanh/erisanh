# Remote Desktop on Arch Linux (Hyprland)

Remote into this machine from elsewhere (e.g. the office) and see the **real,
live Hyprland desktop** — the actual session with every app still open and every
login/cookie intact. Captures the running Hyprland output and serves it as a web
page, gated by a password, reachable from any browser over HTTPS — without
installing anything on the office machine.

```
Office browser
   │  HTTPS + username/password
   ▼
Tailscale Funnel  (public URL: https://<host>.<tailnet>.ts.net)
   ▼
Caddy            127.0.0.1:7000   ← HTTP Basic-Auth gate
   ▼
websockify+noVNC 127.0.0.1:6080   ← web client + VNC⇄WebSocket bridge
   ▼
wayvnc           127.0.0.1:5900   ← captures Hyprland output eDP-1
   ▼
Live Hyprland session
```

## Why these choices

- **Capture `eDP-1` (the laptop's built-in panel), not HDMI.** An external HDMI
  monitor *disconnects* in Hyprland when you power it off, which would kill the
  capture. The internal panel never disconnects.
- **Never use DPMS-off on the captured output.** wlroots stops rendering a
  DPMS-off output, which **freezes** the wayvnc stream. Instead we turn the
  **backlight off** (`brightnessctl set 0`): the panel goes dark (saves power)
  but the compositor keeps rendering, so wayvnc keeps streaming a normal, bright
  image (it reads the framebuffer, not the physical backlight).
- **Everything binds to `127.0.0.1`.** Nothing listens on the LAN; the only
  public entrypoint is Tailscale Funnel, and the only way through it is the Caddy
  password.
- **`python-websockify` is not in the repos** — the `websockify` command comes
  from the **AUR** package `websockify`, so install with `yay`, not `pacman`.

## Files this setup created / changed

- `~/.config/hypr/custom/scripts/remote-desktop.sh` — autostarts wayvnc + noVNC
  (websockify) + Caddy. Re-runnable (kills old copies first).
- `~/.config/hypr/custom/scripts/remote-desktop-setup.sh` — one-time: set the
  login password (Caddyfile), log in to Tailscale, enable Funnel.
- `~/.config/hypr/hyprland/execs.lua` — added a line to run `remote-desktop.sh`
  on Hyprland start.
- `~/.config/hypr/hypridle.conf` — the 10-min idle listener now does
  `brightnessctl -s set 0` / `brightnessctl -r` instead of DPMS off/on.
- `~/noVNC` — cloned noVNC web client.
- `~/.config/caddy/Caddyfile` — generated by the setup script (Basic-Auth gate).

## Setup steps

```bash
# 1. Install (websockify is AUR, so use yay — one command for all four)
yay -S --needed wayvnc tailscale caddy websockify

# 2. One-time: password + Tailscale login + Funnel (interactive)
~/.config/hypr/custom/scripts/remote-desktop-setup.sh
#    - asks for username/password (typed in the browser later)
#    - opens a Tailscale login link
#    - enables Funnel on port 7000 and prints your https URL
#    NOTE: first-time Funnel may require enabling HTTPS + Funnel for the tailnet
#    in the admin console: https://login.tailscale.com/admin

# 3. Start the stack (or just reboot — execs.lua autostarts it)
hyprctl reload
#    or run directly:  ~/.config/hypr/custom/scripts/remote-desktop.sh
```

## Connect from the office

Open the printed URL in any browser:

```
https://<host>.<tailnet>.ts.net/vnc.html
```

Enter the username/password → you see the live Hyprland desktop.

## Leaving the machine at home ("screen off, network on")

- **Keep the lid OPEN.** Closing it disables `eDP-1` (the capture source).
  Letting the backlight go to 0 (idle, or `brightnessctl set 0`) makes the panel
  dark while the remote stream stays bright.
- Power off the **external Acer monitor** with its own button — doesn't matter,
  we capture the internal panel.
- The machine **must not suspend** (auto-suspend is already disabled in
  `hypridle.conf`). Manual sleep/shutdown still works when you're at the machine.
- If you ever *need* the lid closed, switch the capture to a **headless output**
  (`hyprctl output create headless`) so it doesn't depend on a physical screen.

## Troubleshooting

- Blank/frozen remote → the captured output stopped rendering. Check it's not
  DPMS-off: `hyprctl monitors | grep -A1 eDP-1`. Logs: `~/.cache/wayvnc.log`,
  `~/.cache/websockify.log`, `~/.cache/caddy.log`.
- Caddy "basic_auth" vs "basicauth": the setup script auto-detects which your
  Caddy version uses.
- Funnel not reachable → confirm `tailscale funnel status` shows port 7000, and
  that Funnel/HTTPS is enabled for the tailnet in the admin console.
